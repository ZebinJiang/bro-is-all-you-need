## Purpose

Produce a concrete implementation blueprint for a GR00T 1.7 feature, code-input integration, or research-derived idea.

## Output

- patterns and conventions found;
- chosen architecture and rationale;
- files to create or modify in natural official-layout locations;
- component responsibilities and interfaces;
- data/tensor flow;
- Slurm config discovery needs, compute-node debug needs, and dataset implications;
- validation plan;
- complexity, memory, and parallel-efficiency considerations;
- rollback plan.

## Quality bar

Make one clear recommendation when enough information exists. Prefer minimal efficient changes and baseline-safe extension points. Avoid deep wrapper stacks and avoid creating duplicate source trees.
