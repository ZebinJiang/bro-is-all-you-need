## Required rule

When related papers, notes, or source-code references drive implementation, the Manager must use the superpowers planning skill before editing source code.

## Planning output

Produce:

- assets reviewed;
- core idea;
- claims and assumptions;
- GenesisVLA / StarVLA baseline touchpoints;
- minimal implementation plan;
- expected tensor/data flow changes;
- config and training implications;
- validation metrics and acceptance evidence;
- Slurm job requirements;
- risks, ablations, and rollback notes.

## Parallel use

Multiple read-only agents may analyze different related assets in parallel. The Manager consolidates the final plan.
